## Definition of done for shared UI work

<Checklist
  items={[
    <>
      The component uses <strong>semantic tokens</strong> instead of hard-coded brand values.
    </>,
    <>
      Storybook shows the <strong>default state</strong>, meaningful variants, and at least one
      realistic composition example.
    </>,
    <>
      Accessibility concerns such as labels, focus styles, and keyboard interaction are documented
      or visible in stories.
    </>,
    <>
      Relevant docs in <strong>`src/stories/docs/`</strong>, <strong>`README.md`</strong>, and
      <strong>`CLAUDE.md`</strong> stay aligned.
    </>,
  ]}
/>

## Validation checklist

```bash
npm run lint
npm run typecheck
npm run storybook:build
npm run validate:template
```

## Preferred addition path

1. Try to compose the new experience from existing primitives first.
2. If reuse justifies a new primitive, place it in `src/components/ui/`.
3. Add or expand co-located stories immediately.
4. Update style-guide docs if new tokens, patterns, or contribution rules were introduced.
5. Re-run validation before handing the change off.
