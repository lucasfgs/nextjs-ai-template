## Good hierarchy vs noisy hierarchy

<DoDont
  doTitle="Create a predictable reading path"
  doBody={
    <>
      Use one strong heading, one short supporting paragraph, and consistent metadata sizing. This
      keeps cards and settings pages readable even when many sections appear on screen at once.
    </>
  }
  dontTitle="Compete for attention inside one surface"
  dontBody={
    <>
      Avoid multiple oversized headings, decorative capitalization, or mixing muted and
      high-contrast treatments arbitrarily. If every line shouts, nothing becomes easier to scan.
    </>
  }
/>
