## Override pattern

```css
.brand-tenant {
  --primary: #1d4ed8;
  --primary-foreground: #ffffff;
  --ring: #1d4ed8;
  --accent: #dbeafe;
  --accent-foreground: #1e3a8a;
}
```

## Theming guardrails

- Change **semantic tokens**, not one-off component classes, when the brand shifts.
- Keep `--primary`, `--accent`, `--ring`, and any status tones aligned so interactions feel cohesive.
- Validate dark mode after every brand change. A light-theme palette that looks good can still fail in
  dark surfaces or hover states.
- Prefer scoped wrappers for brand variations so downstream apps can run multiple themes without
  mutating the shared global contract.
