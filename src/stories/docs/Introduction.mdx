## Recommended workflow

1. **Choose or add semantic tokens first.** Prefer `bg-primary`, `text-muted-foreground`,
   `shadow-elevation-md`, or `px-layout` over one-off values.
2. **Compose from existing primitives.** Reach for `Card`, `Button`, `Input`, `Alert`, `Dialog`, and
   other shared pieces before introducing new wrappers.
3. **Capture states in Storybook.** Every reusable primitive should show its default, important
   variants, empty/loading/disabled states, and at least one realistic composition example.
4. **Document behavior, not just visuals.** Add accessibility notes, theming implications, and
   contribution guidance when conventions change.
5. **Validate before merging.** Run the repository checks listed in the Contribution Rules page so
   the app, Storybook, and template docs stay aligned.
